1. Wait for the pod to be running:

   kubectl rollout status deployment/{{ include "memguard.fullname" . }} \
     --namespace {{ .Release.Namespace }}

2. Test the endpoints (in a separate terminal, port-forward first):

   kubectl port-forward \
     deployment/{{ include "memguard.fullname" . }} \
     8000:{{ .Values.vllm.port }} 8001:{{ .Values.sidecar.port }} \
     --namespace {{ .Release.Namespace }}

   # vLLM liveness
   curl http://localhost:8000/health

   # memguard liveness (always 200)
   curl http://localhost:8001/healthz

   # memguard readiness (200 = safe, 503 = KV cache above threshold)
   curl -i http://localhost:8001/readyz

3. Example healthy /readyz response:
   {"status": "ready", "oom_probability": 0.12, "threshold": {{ .Values.sidecar.shedThreshold }}}

4. Example /readyz response under OOM pressure:
   {"status": "not_ready", "oom_probability": 0.84, "threshold": {{ .Values.sidecar.shedThreshold }}}
   HTTP/1.1 503 Service Unavailable
